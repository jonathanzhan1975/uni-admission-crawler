## 1. 测试策略

| 层级 | 范围 | 工具 | 占比 |
|---|---|---|---|
| L1 单元测试 | 各模块独立函数 | pytest + 固定快照 | 60% |
| L2 集成测试 | 模块间数据流 | pytest + 真实/录制 HTTP | 25% |
| L3 端到端测试 | 完整管道 | GitHub Actions 试运行 | 10% |
| L4 验收测试 | 7 天连续运行 | 人工 + 自动断言 | 5% |

## 2. 测试数据

### 2.1 固定快照（Fixtures）

仓库内 `tests/fixtures/`：
- `fudan_ao_list_20260501.html`：复旦招办列表页快照
- `sjtu_jwc_20260501.xml`：上交教务 RSS 快照
- `tsinghua_news_20260501.xml`：清华新闻 RSS 快照
- 每个源至少 2 个版本快照（含改版前后对比）

### 2.2 分类金标集（Gold Set）

`tests/fixtures/classification_gold.jsonl`：
- 50 条人工标注样本，标签 `is_admission ∈ {true, false}`
- 来源构成：清华真实新闻 30 条 + 上交教务真实通知 20 条
- **正负比例 30:20**（正样本略多以验证召回）
- **疑难样本占比 ≥ 20%**（如"招生宣传志愿者活动"这种边界情况）

### 3.2 分类器（Classifier）

| 用例 ID | 描述 | 输入 | 期望 | 失败动作 |
|---|---|---|---|---|
| TC-01 | 关键词命中 | "2026 年本科招生简章发布" | 预筛通过 | Block |
| TC-02 | 关键词漏召 | "清华大学迎接 2026 级新生入校" | 预筛**不通过**（应由 LLM 兜底——本用例验证关键词漏召属已知） | 记录但不阻塞 |
| TC-03 | 关键词假阳性 | "招生工作座谈会召开" | 预筛通过 → LLM 应判 false | Block（LLM 也判 true 则需调 prompt） |
| TC-04 | 金标集召回率 | 50 条 gold set 全跑 | recall ≥ 0.90 | Block POC |
| TC-05 | 金标集准确率 | 同上 | precision ≥ 0.85 | Block |
| TC-06 | LLM 不可用降级 | mock API 500 | 降级到关键词，**不抛异常**，标记 degraded=true | Block |
| TC-07 | LLM 单次成本 | 真实调用 100 条 | 免费层内 0 成本；付费层总成本 < $0.01 | Warn |
| TC-08 | 预筛过滤率 | 100 条非招生新闻 | ≤ 20 条进入 LLM | Warn（>20 说明关键词太宽） |

**金标集评估流程**：每次 prompt 改动后必须重跑 TC-04/TC-05，结果记录到 `tests/classification_history.csv`。

### 3.6 端到端（E2E）

| 用例 ID | 描述 | 步骤 | 期望 |
|---|---|---|---|
| TE-01 | 冷启动全流程 | 删除 state.db，运行一次 | 推送成功，state.db 重建，含 N 条记录 |
| TE-02 | 增量运行 | TE-01 后立即再跑 | 推送内容显示 "0 条新消息"，state.db 行数不变 |
| TE-03 | 单源失败隔离 | 把复旦 URL 改成不存在域名 | 上交+清华正常，复旦栏显示"抓取失败" |

| TE-05 | GitHub Actions 试运行 | 手动 trigger workflow | 全流程在 Actions 上跑通，artifact 含日志 |

### 3.7 验收（连续 7 天）

| 用例 ID | 描述 | 验证方法 | 通过条件 |

## 4. 测试执行节奏

| 阶段 | 准入测试 | 准出测试 |

| 开发中 | TF-* 全过、TC-04/05 ≥ 阈值 | — |

## 5. 缺陷分级

| 级别 | 定义 | 处理时限 |
|---|---|---|
| P0 | POC 完全无法推送 | 当日修复 |
| P1 | 单源失效、漏推、单渠道失效 | 2 日内修复 |
| P2 | 误推、格式问题 | 一周内修复 |
| P3 | 优化建议 | 进入 Backlog |

## 6. 测试工具与环境

- **单元/集成**：pytest，本地 Python 3.11
- **HTTP mock**：`responses` 或 `pytest-httpx`
- **快照管理**：手工录制，存入 git LFS（如 > 100KB）
- **CI**：GitHub Actions（与生产同环境）
- **真实抓取测试**：每周一次跑 `pytest -m live`（标记需访问真网络的用例）
