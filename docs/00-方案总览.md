# 方案总览

## 1. 业务目标

每日自动抓取国内 39 所 985 高校的招生相关动态（含本科招生、研究生招生、新增专业、招生政策等），自动过滤分类后推送到飞书群和微信，替代人工逐校浏览官网。

## 2. 技术路线对比

### 2.1 为什么不参考 dailydotdev/daily

dailydotdev 是面向开发者的 PWA 资讯流社交平台（19.8k stars，AGPL-3.0），架构是完整 SaaS：前端 PWA + 浏览器插件 + TypeORM 后端 + Docker。它内容来源是开发者 RSS 源，不涉及中文高校官网。

可借鉴的只有产品形态（personalized feed、每日 digest、按 source 分组），代码层面无可复用。

### 2.2 为什么不抓微信公众号

公众号抓取需要逆向微信读书或公共号 API，技术门槛高、稳定性差、有合规风险。改抓高校官网更直接：信息源权威、HTML 结构稳定、合规（公开页面）。

### 2.3 最终选型

```
[RSSHub 已覆盖高校]      +     [自写 scrapy/bs4 爬虫]
        ↓                                ↓
        └──────────── 统一 Item Schema ────┘
                          ↓
              关键词预筛 + Gemini 2.5 Flash 分类
                          ↓
                  SQLite 去重（item_id = sha256(url)）
                          ↓
                  Jinja2 渲染统一 Markdown
                          ↓
              飞书 Webhook + Server 酱（双渠道）
                          ↓
              GitHub Actions 每日 09:00 cron
```
