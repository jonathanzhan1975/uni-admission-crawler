## 2. 数据契约（schema.py）

所有模块间传递的对象签名。**禁止在模块内部使用 dict 替代这些类型。**

```python
from dataclasses import dataclass
from datetime import datetime
from enum import Enum
from typing import Optional

class SourceId(str, Enum):
    FUDAN_AO       = "S-01"
    FUDAN_GSAO     = "S-02"
    SJTU_JWC       = "S-03"
    SJTU_YZB       = "S-04"
    TSINGHUA_NEWS  = "S-05"

class ChannelId(str, Enum):
    LARK        = "CH-1"
    SERVERCHAN  = "CH-2"

@dataclass(frozen=True)
class Item:
    item_id: str           # = sha256(canonical_url)，主键
    university: str        # 中文全称："复旦大学"
    source_id: SourceId
    source_name: str       # 人类可读："本科招办"
    title: str
    url: str               # canonical 后的绝对 URL
    pub_date: datetime     # 时区敏感
    summary: Optional[str]
    fetched_at: datetime
    date_inferred: bool = False        # FR-2.3
    is_admission: Optional[bool] = None  # 分类结果，未分类的源恒为 True
    classify_reason: Optional[str] = None
    classify_degraded: bool = False    # FR-3.5

@dataclass
class FetchResult:
    source_id: SourceId
    items: list[Item]
    success: bool
    error: Optional[str] = None       # 失败时 items 为空

@dataclass
class PushResult:
    channel: ChannelId
    success: bool
    error: Optional[str] = None
    sent_bytes: int = 0
    retries: int = 0

@dataclass
class RunReport:
    run_id: str                        # GitHub run_id 或本地 timestamp
    started_at: datetime
    fetch_results: list[FetchResult]
    classified_count: int
    deduped_count: int                 # 经去重后剩余
    push_results: list[PushResult]
    classify_degraded: bool
```

### 3.1 `fetchers/base.py`

```python
from abc import ABC, abstractmethod
from .schema import FetchResult, SourceId

class BaseFetcher(ABC):
    source_id: SourceId
    source_name: str
    university: str
    needs_classification: bool

    @abstractmethod
    def fetch(self, max_items: int = 30) -> FetchResult:
        """
        契约：
        - 任何异常**禁止**抛出，必须捕获并返回 success=False 的 FetchResult
        - items 长度 ≤ max_items
        - 返回的每个 Item 必须通过 schema validation
        - 必须调用 archive.save_raw() 落盘原始内容
        """
```

### 3.12 `main.py`

```python
def run(config: AppConfig) -> RunReport:
    """
    主流程：
    1. 实例化 fetchers / classifier / dedup / pushers
    2. 顺序 fetch（独立失败隔离）
    3. classifier.classify(all_items)
    4. dedup.filter_new(classified_items)
    5. render.render(...)
    6. 并发推送到所有 enabled 渠道
    7. 任一渠道成功则 dedup.commit
    8. archive.cleanup(retention_days)
    9. 输出 RunReport JSON 到 stdout
    10. 全渠道失败时 sys.exit(1)
    """

if __name__ == "__main__":
    cfg = load_config()
    report = run(cfg)
    print(report.to_json())
```

## 5. .github/workflows/daily.yml 关键结构

```yaml
name: Daily Crawl
on:
  schedule:
    - cron: '0 1 * * *'        # UTC 01:00 = 北京 09:00
  workflow_dispatch:

jobs:
  run:
    runs-on: ubuntu-latest
    permissions:
      contents: write          # 用于 commit state.db
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: '3.11' }
      - run: pip install -r requirements.txt
      - run: python -m crawler.main
        env:
          LARK_WEBHOOK_URL:   ${{ secrets.LARK_WEBHOOK_URL }}
          SERVERCHAN_SENDKEY: ${{ secrets.SERVERCHAN_SENDKEY }}
          GOOGLE_API_KEY:     ${{ secrets.GOOGLE_API_KEY }}
      - name: Commit state
        run: |
          git config user.name  "github-actions"
          git config user.email "actions@github.com"
          git add data/state.db
          git diff --cached --quiet || git commit -m "chore: update state $(date -u +%F)"
          git push
      - uses: actions/upload-artifact@v4
        if: always()
        with:
          name: run-logs
          path: |
            data/raw/
            data/failed/
```

## 8. 关键不变量（开发期必守，code review 重点检查）

| # | 不变量 | 违反后果 |
|---|---|---|
| I-1 | Fetcher 永不抛异常，全部以 FetchResult 返回 | 单源失败连锁 |
| I-2 | Pusher 永不抛异常，全部以 PushResult 返回 | 推送失败导致 dedup 状态错误 |
| I-3 | dedup.commit 仅在**至少一个渠道推送成功**后调用 | 重复推送或漏推 |
| I-4 | item_id = sha256(canonical_url)，**不**含 title/date | URL 变更检测失效 |
| I-5 | 所有外部 URL 经 `utils/url.canonicalize()` 归一化 | trailing slash 等问题导致重复 |
| I-6 | Secrets 仅从 env 读取，**不**落 config.yaml/state.db/log | 凭证泄露 |
| I-7 | LLM prompt 改动必须重跑金标集，结果记 history.csv | 准确率回归无感知 |
| I-8 | 日志全部走 structlog，禁用 print | E2E 验证依赖结构化日志 |
